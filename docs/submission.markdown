## Submission

### **Model Submission**: 

Please provide your solution to [TBD](). Each team can submit multiple times and we will only use the latest version you submit. Your models and scripts should be accessible and runnable.

#### **Task I**

Participants must submit:
1. **GitHub Repository**: Public or private repo containing your code
2. **HuggingFace Model Link**: Your fine-tuned model uploaded to HuggingFace Hub
3. **Run Script**: A script (e.g., `run.py` or `inference.py`) that we can execute to reproduce your results
4. **Requirements File**: `requirements.txt` with all dependencies

#### **Task II**
Participants must submit:
1. **GitHub Repository**: Public or private repo containing your code
2. **HuggingFace Model Link**: Your fine-tuned model uploaded to HuggingFace Hub
3. **Run Script**: A script (e.g., `run.py` or `inference.py`) that we can execute to reproduce your results
4. **Requirements File**: `requirements.txt` with all dependencies

#### **Task III**
Participants must submit:
1. **GitHub Repository**: Public or private repo containing your trading agent code
2. **HuggingFace Model Link** (if applicable): Any fine-tuned models uploaded to HuggingFace Hub
3. **Run Script**: A script (e.g., `run.py`) that we can execute to run your agent in paper trading simulation
4. **Requirements File**: `requirements.txt` with all dependencies

#### **Task IV**
Participants must submit:
1. **GitHub Repository**: Public or private repository containing the full pipeline used to generate predictions
2. **HuggingFace Model Link** (if applicable): Provide a link if you fine-tuned and uploaded a model to the Hugging Face Hub.
3. **Run Script**: A reproducible script (e.g., `run.py`) that generates predictions end-to-end.
4. **Prediction Output File**  
   - Include your final predictions in the repository.
   - The file format must match the provided example.
   - You may change the prediction output file name, as long as it remains in **CSV** format (`.csv`) and contains the **same columns** as the provided example.



#### **Task V**
Participants must submit the following:

1. **orders JSONL file**: A time-ordered log of all trading actions taken during the evaluation period.
2. **daily equity CSV file**: A record of the portfolio’s daily equity values over the evaluation period.
3. **End-of-period portfolio snapshot**: 
A screenshot of the Alpaca portfolio value at the end of the evaluation period.
